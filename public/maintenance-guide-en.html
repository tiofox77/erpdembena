1. Module Overview

The Maintenance Module of ERP Dembena is a complete solution for managing all aspects of equipment and facility maintenance. The system allows you to:

  • Monitor equipment status in real-time
  • Record and track corrective maintenance
  • Plan preventive maintenance
  • Manage failure modes and causes
  • Categorize maintenance types
  • Generate performance and compliance reports

The module is designed to optimize maintenance processes, reduce equipment downtime, and extend asset lifespan through efficient and systematic monitoring.

In the following sections, you will find detailed instructions on how to use each component of the maintenance module.

2. Maintenance Dashboard

The Maintenance Dashboard provides an overview of equipment health and maintenance process status in a visual and intuitive format.

Fig. 1: Overview of the Maintenance Dashboard

2.1 Using Filters

The dashboard allows you to filter information for specific analysis:

  1. Select the Year to filter data by annual period
  2. Choose the Month to analyze specific monthly data
  3. Filter by Status to view only equipment in a certain state (Operational, In Maintenance, Pending)
  4. Area/Department allows filtering by installation location
  5. Click Update to apply the selected filters

2.2 Key Performance Indicators (KPIs)

The dashboard displays the main maintenance KPIs:

  • Scheduled Maintenance: Total programmed maintenance in the period
  • Completed Maintenance: Number of completed maintenance activities
  • Pending Maintenance: Number of delayed or not-started maintenance activities
  • Compliance Rate: Percentage of maintenance performed on time
  • Mean Time To Repair (MTTR): Average time to complete a maintenance activity
  • Mean Time Between Failures (MTBF): Average interval between failures of the same equipment

2.3 Charts and Visualizations

The dashboard presents different charts to assist in analysis:

  • Bar Chart: Shows maintenance trends over time
  • Pie Chart: Percentage distribution by maintenance type
  • Line Chart: Failure trend over time

Hover over the charts to see specific details for each point or segment.

2.4 Alerts and Notifications

The dashboard displays important alerts:

  • Urgent Alerts: Highlighted in red, requiring immediate attention
  • Upcoming Maintenance: List of scheduled maintenance for the coming days
  • Approaching Deadlines: Maintenance with deadlines close to expiration

Click the Mark as Seen button to acknowledge an alert.

3. Corrective Maintenance

The Corrective Maintenance module allows you to record, track, and manage unscheduled repairs performed after detecting failures or defects in equipment.

Fig. 2: Corrective maintenance management interface

3.1 Recording a New Corrective Maintenance

  1. Click the Record Stoppage button in the upper right corner of the screen
  2. In the form that appears, fill in:
    • Equipment: Select the affected equipment
    • Problem Description: Describe the problem in detail
    • Start Date and Time: When the failure was detected
    • Priority: Define the urgency of repair (Low, Medium, High, Critical)
    • Failure Mode: Select the corresponding failure mode
    • Probable Cause: Indicate the possible cause of the failure
  3. If necessary, add photos or documents by clicking Attach Files
  4. Click Save to record the corrective maintenance

3.2 Filtering and Searching Maintenance Records

To find specific records:

  1. Use the search bar to search by ID, equipment, or description
  2. Use the available filters:
    • Equipment: Filter by specific equipment
    • Status: Filter by Pending, In Progress, Resolved, etc.
    • Period: Select a date range
    • Technician in Charge: Filter by assigned person
  3. Click Apply Filters to update the list

3.3 Updating Status and Information

To update an ongoing maintenance:

  1. Click the Edit button next to the record
  2. Update the Status according to progress: In Progress, Awaiting Parts, Completed, etc.
  3. Add Notes about the actions taken
  4. If completed, fill in:
    • Completion Date and Time
    • Applied Solution: Describe in detail how the problem was resolved
    • Parts Used: List all replaced parts
    • Technician Responsible: Who performed the repair
  5. Click Save to update the record

3.4 Analyzing Maintenance History

To review the complete maintenance history of equipment:

  1. Access the equipment details page
  2. In the Maintenance History section, you'll see all previous records
  3. Use date filters to refine the search
  4. Click Export History to generate a report in PDF or Excel

Use this feature to identify failure patterns and plan preventive maintenance.

Tip: Keep corrective maintenance records always updated, including as much detail as possible. Complete information helps identify root causes and optimize maintenance processes.

4. Equipment Registry

The Equipment Registry module allows you to manage the registration and information of all assets that require maintenance, facilitating control and monitoring.

Fig. 3: Equipment registration and management interface

4.1 Registering a New Equipment

  1. Access the Equipment page in the side menu
  2. Click the New Equipment button
  3. Fill in the required fields:
    • Equipment Code: Unique identifier
    • Name: Description/name of the equipment
    • Model: Model or type
    • Manufacturer: Manufacturing company
    • Serial Number: Equipment serial number
    • Department/Area: Where it is installed
    • Acquisition Date: When it was acquired
    • Current Status: Operational, In Maintenance, Inactive, etc.
  4. Fill in additional fields when applicable:
    • Technical Specifications: Technical details
    • Warranty: Warranty period and details
    • Documents: Manuals and certificates
    • Supplier: Supplying company
    • Acquisition Value: Equipment cost
    • Estimated Useful Life: Expected operation time
  5. Add a photo or image of the equipment by clicking Add Image
  6. Click Save to create the record

4.2 Managing Components and Parts

To register the components of equipment:

  1. Access the equipment details page
  2. Go to the Components tab
  3. Click Add Component
  4. Fill in the component details:
    • Component Name
    • Type
    • Quantity
    • Description
    • Part Number (if applicable)
  5. Define if the component is Critical (directly affects operation)
  6. Click Save Component

This facilitates the management of replacement parts and specific maintenance for each component.

4.3 Managing Technical Documents

To attach technical documentation to the equipment:

  1. On the equipment details page, access the Documents tab
  2. Click Attach Document
  3. Select the Document Type: Manual, Certificate, Electrical Diagram, etc.
  4. Add a Description to identify it
  5. Click Choose File and select the document to be uploaded
  6. Click Submit to complete

The documents will be available for consultation during maintenance activities.

4.4 Viewing Equipment History

To consult the complete history of equipment:

  1. On the equipment details page, access the History tab
  2. View all previous maintenance activities, categorized by type:
    • Corrective Maintenance
    • Preventive Maintenance
    • Calibrations
    • Inspections
  3. Use date and type filters to refine the search
  4. Click on any record to see the complete details

The history allows reliability analysis and identification of recurring failure patterns.

Attention: Keep equipment information always up-to-date, especially supplier contact information and warranty details. This is crucial for expediting repairs and activating warranties when necessary.

5. Maintenance Planning

The Maintenance Planning module allows you to create and manage preventive maintenance plans, establishing systematic routines for inspection, lubrication, adjustments, and part replacement before failures occur.

Fig. 4: Preventive maintenance planning interface

5.1 Creating a New Maintenance Plan

  1. In the side menu, access Maintenance Planning
  2. Click the New Plan button
  3. Fill in the main information:
    • Plan Title: Descriptive name
    • Description: Details about the plan's objective
    • Equipment(s): Select the equipment to be included
    • Responsible: Technician or team responsible
  4. Define the Periodicity of the plan:
    • Daily: For quick inspections or lubrications
    • Weekly: For routine checks
    • Monthly: For medium complexity maintenance
    • Quarterly/Biannual/Annual: For more complete maintenance
    • Custom: Based on operation hours or cycles
  5. Configure Notifications to alert in advance about scheduled maintenance
  6. Click Next to continue

5.2 Adding Activities to the Plan

After creating the plan, add the activities to be performed:

  1. In the Activities section, click Add Activity
  2. For each activity, fill in:
    • Activity Description: What needs to be done
    • Estimated Time: Expected duration
    • Skill Level: Required technical skill
    • Required Tools: Equipment for execution
    • Safety Procedures: Specific precautions
    • Detailed Procedure: Step-by-step execution
  3. Add Images or Diagrams to illustrate the procedure, if necessary
  4. Click Save Activity and repeat to add more activities
  5. When finished, click Complete Plan

5.3 Viewing the Maintenance Calendar

To view all scheduled maintenance on a calendar:

  1. On the Planning page, click the Calendar tab
  2. Select the desired View: Monthly, Weekly, or Daily
  3. Use filters to show only:
    • Maintenance for specific equipment
    • Maintenance assigned to a technician
    • Specific type of maintenance
  4. Hover over an event to see quick details
  5. Click on an event to see complete details or edit

The calendar provides a clear visualization of task distribution over time.

5.4 Executing a Planned Maintenance

When the time comes to execute a scheduled maintenance:

  1. Access the Work Orders page
  2. Locate the WO automatically generated by the maintenance plan
  3. Click Start Maintenance
  4. Follow the checklist for each activity:
    • Mark each item as Completed after execution
    • Record Observations about anomalies found
    • Record Measurements taken, when applicable
  5. Record the Parts Used, if components were replaced
  6. Add Photos of before and after, if relevant
  7. When finished, click Complete Maintenance

The system will automatically update the equipment history and schedule the next maintenance according to the configured periodicity.

Tip: Well-planned preventive maintenance significantly reduces the occurrence of unexpected failures and extends the equipment's useful life. It also reduces operational costs and unscheduled downtime.

6. Maintenance Categories

The Maintenance Categories module allows you to classify and organize the different types of maintenance performed on equipment, facilitating planning, resource allocation, and performance analysis.

Fig. 5: Maintenance categories management interface

6.1 Predefined Category Types

The system offers some predefined categories that cover the main types of maintenance:

Category Description Visual Identification
Preventive Maintenance performed at predefined intervals to prevent failures Preventive
Corrective Maintenance performed after failures occur to restore functionality Corrective
Predictive Maintenance based on analysis and monitoring of equipment conditions Predictive
Detection Inspections and tests to identify potential problems Detection
Lubrication Application of lubricants to mechanical components Lubrication

6.2 Creating New Categories

To create a new custom maintenance category:

  1. In the side menu, access Settings and then Maintenance Categories
  2. Click the New Category button
  3. Fill in the required fields:
    • Category Name: Clear and concise identification
    • Description: Detailed explanation of the maintenance type
    • Color: Choose a color to visually identify the category
    • Default Priority: Typical urgency level (Low, Medium, High, Critical)
    • Average Estimated Time: Typical duration of this type of maintenance
  4. Define Permissions for access by different user profiles
  5. Click Save to create the new category

6.3 Managing Existing Categories

To manage existing categories:

  • Edit: Click the edit icon next to the desired category
  • Deactivate: To discontinue a category without completely deleting it
  • Delete: Only possible if the category is not being used in any record
  • Merge: Combine two similar categories, transferring all records from one to another

Note: Modifying categories does not affect historical records already classified.

6.4 Analyzing Data by Category

Use category classification for data analysis:

  • Access Reports > Analysis by Category
  • View comparative charts and statistics between different maintenance types
  • Identify which categories consume more resources
  • Compare average execution times between similar categories
  • Analyze the evolution of the volume of each category over time

This analysis helps identify trends and opportunities for improvement in maintenance processes.

Recommendation: Maintain a limited number of categories to facilitate classification and analysis. Too many categories can make the system confusing. Prioritize categories that actually provide valuable insights for maintenance management.

7. Failure Modes

The Failure Modes module allows you to document and manage the different ways equipment can present problems or failures, facilitating efficient diagnosis and problem resolution.

Fig. 6: Failure modes management interface

7.1 Understanding Failure Modes

A failure mode is the specific way in which equipment or a component fails or ceases to fulfill its intended function. Common examples include:

  • Leakage
  • Breakage
  • Clogging
  • Misalignment
  • Wear
  • Short-circuit
  • Excessive vibration
  • Overheating

Documenting failure modes helps standardize diagnosis and facilitates the identification of recurring failure patterns.

7.2 Registering a New Failure Mode

  1. In the side menu, access Settings and then Failure Modes
  2. Click the New Failure Mode button
  3. Fill in the form fields:
    • Failure Mode Name: Concise description
    • Detailed Description: Complete explanation
    • Category: Select the failure mode category
    • Severity: Impact level (Low, Medium, High, Critical)
    • Common Symptoms: Observable signs that indicate this failure mode
    • Probable Causes: Select the frequently associated causes
  4. Add illustrative Images if necessary
  5. Define Recommended Actions for diagnosis and repair
  6. Click Save to register the new failure mode

7.3 Associating Failure Modes with Equipment

To associate specific failure modes with certain equipment:

  1. Access the equipment details page
  2. Go to the Failure Modes tab
  3. Click Associate Failure Mode
  4. Select the failure modes applicable to this equipment
  5. For each failure mode, define:
    • Probability: Chance of occurrence in this equipment
    • Detectability: Ease of detection before complete failure
    • Average Repair Time: Estimate to resolve this type of failure
  6. Click Save Associations

This association allows faster and more accurate selection during failure recording.

7.4 Consulting the Failure Modes Library

To consult the complete failure modes library:

  1. Access Library > Failure Modes
  2. Use filters to refine the search:
    • Category: Filter by failure mode type
    • Severity: Filter by impact level
    • Equipment: See failure modes associated with specific equipment
  3. Use the Search to find failure modes by keyword
  4. Click on any failure mode to see its complete description, probable causes, and recommended solutions

This library serves as a knowledge base for training and quick reference during problem diagnosis.

Best Practices: Establish a process for periodic review of registered failure modes, updating them based on recent experiences and adding new ones as needed. A well-documented failure modes library is a valuable asset for the organization's technical knowledge management.

9. Failure Causes

The Failure Causes module allows you to identify and document the origins or fundamental reasons that lead to failure modes, facilitating the implementation of effective corrective and preventive actions.

9.1 Differentiating Failure Modes and Causes

It's important to understand the difference between failure modes and failure causes:

  • Failure Mode: Is how the failure manifests, that is, the way the equipment presents the problem (e.g., leakage, breakage, overheating)
  • Failure Cause: Is the factor that originated the failure mode, that is, the underlying reason that led to the problem (e.g., corrosion, material fatigue, operational error)

For example, a failure mode might be "Rupture of a hydraulic hose," while the failure cause might be "Wear by abrasion due to friction with a metal surface."

9.2 Registering a New Failure Cause

  1. In the side menu, access Settings and then Failure Causes
  2. Click the New Failure Cause button
  3. Fill in the form fields:
    • Cause Name: Concise identification
    • Detailed Description: Complete explanation of the failure mechanism
    • Category: Select the failure cause category
    • Detectability: Ease of identification (Easy, Medium, Difficult)
    • Associated Failure Modes: Which failure modes can be caused by this cause
  4. Add Recommended Preventive Actions to avoid this failure cause
  5. Click Save to register the new failure cause

9.3 Root Cause Analysis

The system allows you to perform an in-depth analysis to identify the root cause of recurring problems:

  1. On the Corrective Maintenance page, select a record and click Root Cause Analysis
  2. Use the 5 Whys tool:
    • Why did the problem occur? (1st level)
    • Why did this cause occur? (2nd level)
    • Why was this condition present? (3rd level)
    • Why wasn't it detected earlier? (4th level)
    • Why are there no controls? (5th level)
  3. For each answer, record the corresponding cause and classify it
  4. Create an Action Plan to eliminate the identified root cause
  5. Establish Monitoring Indicators to verify the effectiveness of the actions

This structured methodology allows systematic identification of fundamental causes and their permanent elimination.

9.4 Cause-Effect Relationship Matrix

The system automatically generates a matrix that relates causes and failure modes:

  1. Access Reports > Cause-Effect Matrix
  2. Select the Period and Equipment to be analyzed
  3. The matrix will show which causes are most frequently associated with certain failure modes
  4. Identify critical causes that trigger multiple failure modes
  5. Prioritize preventive actions for the most impactful causes

This analysis allows directing efforts to eliminate the causes that provoke the greatest number of problems.

Best Practices: When identifying failure causes, always seek the most fundamental one possible. For example, instead of just recording "lubrication failure," investigate why there was a lubrication failure: was it due to forgetfulness, a procedure error, lack of training, or another more specific reason? The deeper the investigation, the more effective the corrective action will be.

10. Failure Cause Categories

The Failure Cause Categories module allows you to classify the different origins of failures into logical groups, facilitating the identification of patterns and implementation of systematic improvement actions.

10.1 Standard Failure Cause Categories

The system offers some predefined categories for classifying failure causes:

Category Description Cause Examples
Natural Wear Normal deterioration due to use over time Material fatigue, aging, end of useful life
Operational Error Failures caused by incorrect equipment operation Overload, operation outside parameters, handling error
Maintenance Failure Problems arising from inadequate maintenance Insufficient lubrication, incorrect adjustment, poorly executed repair
Environmental Factors Causes related to the operating environment Extreme temperature, humidity, dirt, external vibration
Design Failure Problems originating from errors in equipment design Undersizing, inadequate material selection, improper design
Material Quality Failures related to component quality Manufacturing defect, out-of-spec material, counterfeit

10.2 Creating a New Failure Cause Category

  1. In the side menu, access Settings and then Failure Cause Categories
  2. Click the New Category button
  3. Fill in the required fields:
    • Category Name: Clear and concise identification
    • Description: Detailed explanation of the cause type
    • Color: Choose a color for visual identification of the category
    • Responsible Area: Department normally responsible for actions related to this category
  4. Click Save to create the new category

10.3 Analyzing Data by Cause Category

To gain insights about the most frequent failure cause categories:

  1. Access Reports > Cause Analysis
  2. Select the desired period and the equipment (or group of equipment)
  3. View the cause distribution chart by category
  4. Identify the most recurring categories
  5. For each critical category, deepen the analysis by exploring the specific causes
  6. Establish targeted action plans for the most impactful categories

This analysis allows implementing systematic actions that address entire groups of related causes.

10.4 Improvement Strategies by Category

Each failure cause category requires different approaches to mitigate problems:

Category Improvement Strategies
Natural Wear Preventive replacement plans, condition monitoring, specification of more durable materials
Operational Error Training, creation of standard operating procedures, mistake-proofing systems (poka-yoke)
Maintenance Failure Procedure revision, technical team training, verification checklist
Environmental Factors Environment adaptation, additional protections, selection of components resistant to local conditions
Design Failure Reengineering, design modifications, manufacturer involvement

By properly categorizing causes, it's possible to implement comprehensive improvement programs that address entire groups of similar problems.

Recommendation: Periodically perform a Pareto analysis of failure cause categories to identify which ones are responsible for most of the problems (typically following the 80/20 principle, where 20% of cause categories are responsible for 80% of failures). Focus your improvement efforts on these critical categories to maximize return on investment.

11. Maintenance Reports

The Maintenance Reports module provides tools for data analysis, performance monitoring, and decision-making based on consolidated information about all maintenance activities.

Fig. 7: Maintenance reports module interface

11.1 Available Reports

The system offers various pre-configured reports for data analysis:

  • Maintenance Summary: Overview of all maintenance performed in a period
  • Equipment Performance: Analysis of availability, reliability, and costs per equipment
  • Maintenance Indicators: MTBF, MTTR, availability, and other KPIs
  • Failure Analysis: Distribution of failures by mode, cause, and category
  • Maintenance Costs: Breakdown of costs by type, equipment, and period
  • Equipment History: Complete record of interventions by equipment
  • Schedule and Compliance: Tracking of preventive maintenance plans
  • Parts Inventory: Consumption and availability of replacement parts

11.2 Filtering and Customizing Reports

To obtain more specific information:

  1. Access the Reports module
  2. Select the desired report type
  3. Use the available filters to refine the data:
    • Period: Select the date range to be analyzed
    • Equipment/Group: Filter by specific equipment or group
    • Department/Area: Restrict to equipment location
    • Maintenance Type: Filter by category (preventive, corrective, etc.)
    • Responsible: Filter by technician or team
  4. Click Generate Report to apply the filters
  5. Use the Customization options to adjust the visualization:
    • Choose the chart type (bar, pie, line, etc.)
    • Select the columns to be displayed in tables
    • Define the data grouping (by month, by category, etc.)
    • Adjust the scale and colors of charts

Custom reports can be saved for future use.

11.3 Exporting and Sharing Reports

After generating a report, you can export it in different formats:

  • PDF: For presentations and formal documents
  • Excel: For additional data analysis
  • CSV: For import into other systems
  • Image (PNG/JPG): For inclusion in presentations

To share reports with the team:

  1. Click the Share button
  2. Select the sharing method:
    • Email: Send the report directly by email
    • Link: Generate a link for report access
    • Schedule: Configure periodic automatic sending
  3. Define recipients and access permissions
  4. Add comments or observations, if necessary

11.4 Configuring KPIs and Dashboards

Create custom dashboards with the most relevant indicators for your operation:

  1. Access Settings > Custom Dashboards
  2. Click New Dashboard
  3. Select the KPIs you want to monitor, such as:
    • Mean Time Between Failures (MTBF)
    • Mean Time To Repair (MTTR)
    • Equipment Availability
    • Preventive Compliance Rate
    • Maintenance Cost to Replacement Value (%)
    • Maintenance Backlog (hours)
  4. Configure Alerts for notification when indicators exceed predefined limits
  5. Set the Periodicity of data updates
  6. Organize the elements in the dashboard layout using the drag-and-drop interface
  7. Save the dashboard and set as home page, if desired

11.5 Trend Analysis and Predictions

Use advanced analysis tools to identify trends and make predictions:

  • Trend Analysis: Visualize the evolution of indicators over time
  • Correlation: Identify relationships between different variables (e.g., ambient temperature vs. failures)
  • Failure Prediction: Based on history, estimate when the next failures will occur
  • Simulation: Test different maintenance scenarios and their impacts
  • Reliability Analysis: Assess equipment reliability over time

These tools help anticipate problems and optimize maintenance strategies.

Tip: Regular analysis of maintenance reports is essential for continuous improvement. Establish periodic meetings to review indicators, identify improvement opportunities, and define corrective actions. Remember that quality data is essential for reliable analyses, so encourage accurate and complete recording of all maintenance activities.
