1. Visão Geral do Módulo

O módulo de Manutenção do ERP Dembena é uma solução completa para gerenciar todos os aspectos da manutenção de equipamentos e instalações. O sistema permite:

  • Monitorar o status de equipamentos em tempo real
  • Registrar e acompanhar manutenções corretivas
  • Planejar manutenções preventivas
  • Gerenciar modos e causas de falhas
  • Categorizar tipos de manutenção
  • Gerar relatórios de desempenho e conformidade

O módulo foi projetado para otimizar os processos de manutenção, reduzir o tempo de inatividade dos equipamentos e prolongar sua vida útil através de um acompanhamento eficiente e sistemático.

Nas seções a seguir, você encontrará instruções detalhadas sobre como utilizar cada componente do módulo de manutenção.

2. Dashboard de Manutenção

O Dashboard de Manutenção oferece uma visão geral da saúde e status dos equipamentos e processos de manutenção em um formato visual e intuitivo.

Fig. 1: Visão geral do Dashboard de Manutenção

2.1 Utilizando os Filtros

O dashboard permite filtrar informações para análise específica:

  1. Selecione o Ano para filtrar os dados por período anual
  2. Escolha o Mês para analisar dados mensais específicos
  3. Filtre por Status para visualizar apenas os equipamentos em determinado estado (Operacional, Em Manutenção, Pendente)
  4. Área/Departamento permite filtrar por local de instalação
  5. Clique em Atualizar para aplicar os filtros selecionados

2.2 Indicadores Chave de Desempenho (KPIs)

O dashboard exibe os principais KPIs de manutenção:

  • Manutenções Previstas: Total de manutenções programadas no período
  • Manutenções Concluídas: Quantidade de manutenções finalizadas
  • Manutenções Pendentes: Número de manutenções em atraso ou não iniciadas
  • Índice de Conformidade: Percentual de manutenções realizadas dentro do prazo
  • Tempo Médio de Reparo (MTTR): Tempo médio para conclusão de uma manutenção
  • Tempo Médio Entre Falhas (MTBF): Intervalo médio entre falhas do mesmo equipamento

2.3 Gráficos e Visualizações

O dashboard apresenta diferentes gráficos para auxiliar na análise:

  • Gráfico de Barras: Mostra a tendência de manutenções ao longo do tempo
  • Gráfico de Pizza: Distribuição percentual por tipo de manutenção
  • Gráfico de Linha: Tendência de falhas ao longo do tempo

Passe o mouse sobre os gráficos para ver detalhes específicos de cada ponto ou segmento.

2.4 Alertas e Notificações

O dashboard exibe alertas importantes:

  • Alertas Urgentes: Destacados em vermelho, requerem atenção imediata
  • Próximas Manutenções: Lista de manutenções programadas para os próximos dias
  • Vencimentos: Manutenções com prazo próximo do vencimento

Clique no botão Marcar como Visto para reconhecer um alerta.

3. Manutenção Corretiva

O módulo de Manutenção Corretiva permite registrar, acompanhar e gerenciar os reparos não programados realizados após a detecção de falhas ou defeitos em equipamentos.

Fig. 2: Interface de gerenciamento de manutenção corretiva

3.1 Registrando uma Nova Manutenção Corretiva

  1. Clique no botão Registrar Parada no canto superior direito da tela
  2. No formulário que aparecer, preencha:
    • Equipamento: Selecione o equipamento afetado
    • Descrição do Problema: Descreva detalhadamente o problema
    • Data e Hora de Início: Quando a falha foi detectada
    • Prioridade: Defina a urgência do reparo (Baixa, Média, Alta, Crítica)
    • Modo de Falha: Selecione o modo de falha correspondente
    • Causa Provável: Indique a possível causa da falha
  3. Se necessário, adicione fotos ou documentos clicando em Anexar Arquivos
  4. Clique em Salvar para registrar a manutenção corretiva

3.2 Filtrando e Pesquisando Manutenções

Para encontrar registros específicos:

  1. Utilize a barra de pesquisa para buscar por ID, equipamento ou descrição
  2. Use os filtros disponíveis:
    • Equipamento: Filtre por equipamento específico
    • Status: Filtre por Pendente, Em Andamento, Resolvido, etc.
    • Período: Selecione um intervalo de datas
    • Técnico Responsável: Filtre por pessoa designada
  3. Clique em Aplicar Filtros para atualizar a lista

3.3 Atualizando Status e Informações

Para atualizar uma manutenção em andamento:

  1. Clique no botão Editar ao lado do registro
  2. Atualize o Status conforme o andamento: Em Andamento, Aguardando Peças, Concluído, etc.
  3. Adicione Anotações sobre as ações realizadas
  4. Se concluído, preencha:
    • Data e Hora de Conclusão
    • Solução Aplicada: Descreva em detalhes como o problema foi resolvido
    • Peças Utilizadas: Liste todas as peças substituídas
    • Técnico Responsável: Quem executou o reparo
  5. Clique em Salvar para atualizar o registro

3.4 Analisando Histórico de Manutenções

Para analisar o histórico de manutenções de um equipamento:

  1. Acesse a página de detalhes do equipamento
  2. Na seção Histórico de Manutenções, você verá todos os registros anteriores
  3. Utilize os filtros de data para refinar a busca
  4. Clique em Exportar Histórico para gerar um relatório em PDF ou Excel

Use esta funcionalidade para identificar padrões de falha e planejar manutenções preventivas.

Dica: Mantenha os registros de manutenção corretiva sempre atualizados, incluindo o máximo de detalhes possível. Informações completas auxiliam na identificação de causas-raiz e na otimização dos processos de manutenção.

4. Cadastro de Equipamentos

O módulo de Cadastro de Equipamentos permite gerenciar o registro e as informações de todos os ativos que necessitam de manutenção, facilitando o controle e o acompanhamento.

Fig. 3: Interface de cadastro e gerenciamento de equipamentos

4.1 Cadastrando um Novo Equipamento

  1. Acesse a página Equipamentos no menu lateral
  2. Clique no botão Novo Equipamento
  3. Preencha os campos obrigatórios:
    • Código do Equipamento: Identificador único
    • Nome: Descrição/nome do equipamento
    • Modelo: Modelo ou tipo
    • Fabricante: Empresa fabricante
    • Número de Série: Número de série do equipamento
    • Departamento/Área: Local onde está instalado
    • Data de Aquisição: Quando foi adquirido
    • Status Atual: Operacional, Em Manutenção, Inativo, etc.
  4. Preencha os campos adicionais quando aplicável:
    • Especificações Técnicas: Detalhes técnicos
    • Garantia: Período e detalhes da garantia
    • Documentos: Manuais e certificados
    • Fornecedor: Empresa fornecedora
    • Valor de Aquisição: Custo do equipamento
    • Vida Útil Estimada: Tempo esperado de operação
  5. Adicione uma foto ou imagem do equipamento clicando em Adicionar Imagem
  6. Clique em Salvar para criar o registro

4.2 Gerenciando Componentes e Peças

Para registrar os componentes de um equipamento:

  1. Acesse a página de detalhes do equipamento
  2. Vá para a aba Componentes
  3. Clique em Adicionar Componente
  4. Preencha os detalhes do componente:
    • Nome do Componente
    • Tipo
    • Quantidade
    • Descrição
    • Código da Peça (se aplicável)
  5. Defina se o componente é Crítico (afeta diretamente a operação)
  6. Clique em Salvar Componente

Isso facilita o gerenciamento de peças de reposição e manutenções específicas para cada componente.

4.3 Gerenciando Documentos Técnicos

Para anexar documentação técnica ao equipamento:

  1. Na página de detalhes do equipamento, acesse a aba Documentos
  2. Clique em Anexar Documento
  3. Selecione o Tipo de Documento: Manual, Certificado, Esquema Elétrico, etc.
  4. Adicione uma Descrição para identificá-lo
  5. Clique em Escolher Arquivo e selecione o documento a ser carregado
  6. Clique em Enviar para concluir

Os documentos ficarão disponíveis para consulta durante as atividades de manutenção.

4.4 Visualizando Histórico do Equipamento

Para consultar o histórico completo de um equipamento:

  1. Na página de detalhes do equipamento, acesse a aba Histórico
  2. Visualize todas as manutenções anteriores, categorias por tipo:
    • Manutenções Corretivas
    • Manutenções Preventivas
    • Calibrações
    • Inspeções
  3. Use os filtros de data e tipo para refinar a busca
  4. Clique em qualquer registro para ver os detalhes completos

O histórico permite análise de confiabilidade e identificação de padRões de falha recorrentes.

Atenção: Mantenha as informações dos equipamentos sempre atualizadas, especialmente dados de contato do fornecedor e detalhes de garantia. Isso é crucial para agilizar reparos e acionar garantias quando necessário.

5. Planejamento de Manutenção

O módulo de Planejamento de Manutenção permite criar e gerenciar planos de manutenção preventiva, estabelecendo rotinas sistemáticas para inspeção, lubrificação, ajustes e substituição de peças, antes que ocorram falhas.

Fig. 4: Interface de planejamento de manutenção preventiva

5.1 Criando um Novo Plano de Manutenção

  1. No menu lateral, acesse Planejamento de Manutenção
  2. Clique no botão Novo Plano
  3. Preencha as informações principais:
    • Título do Plano: Nome descritivo
    • Descrição: Detalhes sobre o objetivo do plano
    • Equipamento(s): Selecione os equipamentos que serão incluídos
    • Responsável: Técnico ou equipe responsável
  4. Defina a Periodicidade do plano:
    • Diária: Para inspeções rápidas ou lubrificações
    • Semanal: Para verificações de rotina
    • Mensal: Para manutenções de média complexidade
    • Trimestral/Semestral/Anual: Para manutenções mais completas
    • Personalizada: Baseada em horas de operação ou ciclos
  5. Configure Notificações para avisar com antecedência sobre as manutenções programadas
  6. Clique em Próximo para continuar

5.2 Adicionando Atividades ao Plano

Após criar o plano, adicione as atividades a serem executadas:

  1. Na seção Atividades, clique em Adicionar Atividade
  2. Para cada atividade, preencha:
    • Descrição da Atividade: O que deve ser feito
    • Tempo Estimado: Duração prevista
    • Nível de Especialização: Habilidade técnica necessária
    • Ferramentas Necessárias: Equipamentos para execução
    • Procedimentos de Segurança: Precauções específicas
    • Procedimento Detalhado: Passo a passo da execução
  3. Adicione Imagens ou Diagramas para ilustrar o procedimento, se necessário
  4. Clique em Salvar Atividade e repita para adicionar mais atividades
  5. Ao concluir, clique em Finalizar Plano

5.3 Visualizando o Calendário de Manutenções

Para visualizar todas as manutenções programadas em um calendário:

  1. Na página de Planejamento, clique na aba Calendário
  2. Selecione a Visualização desejada: Mensal, Semanal ou Diária
  3. Utilize os filtros para exibir apenas:
    • Manutenções de equipamentos específicos
    • Manutenções atribuídas a um técnico
    • Determinado tipo de manutenção
  4. Passe o mouse sobre um evento para ver detalhes rápidos
  5. Clique em um evento para ver detalhes completos ou editar

O calendário permite uma visualização clara da distribuição de tarefas ao longo do tempo.

5.4 Executando uma Manutenção Planejada

Quando chegar o momento de executar uma manutenção programada:

  1. Acesse a página Ordens de Serviço
  2. Localize a OS gerada automaticamente pelo plano de manutenção
  3. Clique em Iniciar Manutenção
  4. Siga a lista de verificação (checklist) para cada atividade:
    • Marque cada item como Concluído após a execução
    • Registre Observações sobre anomalias encontradas
    • Registre Medições realizadas, quando aplicável
  5. Registre as Peças Utilizadas, se houver substituição de componentes
  6. Adicione Fotos do antes e depois, se relevante
  7. Ao finalizar, clique em Concluir Manutenção

O sistema atualizará automaticamente o histórico do equipamento e agendará a próxima manutenção conforme a periodicidade configurada.

Dica: A manutenção preventiva bem planejada reduz significativamente a ocorrência de falhas inesperadas e prolonga a vida útil dos equipamentos. Além disso, reduz custos operacionais e tempos de parada não programada.

6. Categorias de Manutenção

O módulo de Categorias de Manutenção permite classificar e organizar os diferentes tipos de manutenção realizados nos equipamentos, facilitando o planejamento, a alocação de recursos e a análise de desempenho.

Fig. 5: Interface de gerenciamento de categorias de manutenção

6.1 Tipos de Categorias Pré-definidas

O sistema oferece algumas categorias pré-definidas que abrangem os principais tipos de manutenção:

Categoria Descrição Identificação Visual
Preventiva Manutenções realizadas em intervalos pré-definidos para prevenir falhas Preventiva
Corretiva Manutenções realizadas após a ocorrência de falhas para restaurar a funcionalidade Corretiva
Preditiva Manutenções baseadas na análise e monitoramento de condições do equipamento Preditiva
Detecção Inspeções e testes para identificar problemas potenciais Detecção
Lubrificação Aplicação de lubrificantes em componentes mecânicos Lubrificação

6.2 Criando Novas Categorias

Para criar uma nova categoria de manutenção personalizada:

  1. No menu lateral, acesse Configurações e depois Categorias de Manutenção
  2. Clique no botão Nova Categoria
  3. Preencha os campos necessários:
    • Nome da Categoria: Identificação clara e concisa
    • Descrição: Explicação detalhada do tipo de manutenção
    • Cor: Escolha uma cor para identificar visualmente a categoria
    • Prioridade Padrão: Nível de urgência típico (Baixa, Média, Alta, Crítica)
    • Tempo Médio Estimado: Duração típica desse tipo de manutenção
  4. Defina as Permissões de acesso para diferentes perfis de usuários
  5. Clique em Salvar para criar a nova categoria

6.3 Gerenciando Categorias Existentes

Para gerenciar categorias existentes:

  • Editar: Clique no ícone de edição ao lado da categoria desejada
  • Desativar: Para descontinuar uma categoria sem excluí-la completamente
  • Excluir: Apenas possível se a categoria não estiver sendo utilizada em nenhum registro
  • Mesclar: Combine duas categorias similares, transferindo todos os registros de uma para outra

Nota: A modificação de categorias não afeta os registros históricos já classificados.

6.4 Analisando Dados por Categoria

Use a classificação por categorias para análise de dados:

  • Acesse Relatórios > Análise por Categoria
  • Visualize gráficos e estatísticas comparativas entre diferentes tipos de manutenção
  • Identifique quais categorias consomem mais recursos
  • Compare tempos médios de execução entre categorias similares
  • Analise a evolução do volume de cada categoria ao longo do tempo

Esta análise ajuda a identificar tendências e oportunidades de melhoria nos processos de manutenção.

Recomendação: Mantenha um número limitado de categorias para facilitar a classificação e análise. Muitas categorias podem tornar o sistema confuso. Priorize categorias que realmente fornecem insights valiosos para a gestão da manutenção.

7. Modos de Falha

O módulo de Modos de Falha permite documentar e gerenciar as diferentes formas como os equipamentos podem apresentar problemas ou falhas, facilitando o diagnóstico e a resolução eficiente de problemas.

Fig. 6: Interface de gerenciamento de modos de falha

7.1 Entendendo os Modos de Falha

Um modo de falha é a maneira específica como um equipamento ou componente falha ou deixa de cumprir sua função pretendida. Exemplos comuns incluem:

  • Vazamento
  • Quebra
  • Entupimento
  • Desalinhamento
  • Desgaste
  • Curto-circuito
  • Vibração excessiva
  • Superaquecimento

Documentar os modos de falha ajuda a padronizar o diagnóstico e facilita a identificação de padrões recorrentes de falha.

7.2 Cadastrando um Novo Modo de Falha

  1. No menu lateral, acesse Configurações e depois Modos de Falha
  2. Clique no botão Novo Modo de Falha
  3. Preencha os campos do formulário:
    • Nome do Modo de Falha: Descrição concisa
    • Descrição Detalhada: Explicação completa
    • Categoria: Selecione a categoria do modo de falha
    • Severidade: Nível de impacto (Baixo, Médio, Alto, Crítico)
    • Sintomas Comuns: Sinais observáveis que indicam este modo de falha
    • Causas Prováveis: Selecione as causas frequentemente associadas
  4. Adicione Imagens ilustrativas se necessário
  5. Defina Ações Recomendadas para diagnóstico e reparo
  6. Clique em Salvar para registrar o novo modo de falha

7.3 Associando Modos de Falha a Equipamentos

Para associar modos de falha específicos a determinados equipamentos:

  1. Acesse a página de detalhes do equipamento
  2. Vá para a aba Modos de Falha
  3. Clique em Associar Modo de Falha
  4. Selecione os modos de falha aplicáveis a este equipamento
  5. Para cada modo de falha, defina:
    • Probabilidade: Chance de ocorrência neste equipamento
    • Detectabilidade: Facilidade de detecção antes da falha completa
    • Tempo Médio de Reparo: Estimativa para resolver este tipo de falha
  6. Clique em Salvar Associações

Esta associação permite uma seleção mais rápida e precisa durante o registro de falhas.

7.4 Consultando a Biblioteca de Modos de Falha

Para consultar a biblioteca completa de modos de falha:

  1. Acesse Biblioteca > Modos de Falha
  2. Utilize os filtros para refinar a busca:
    • Categoria: Filtre por tipo de modo de falha
    • Severidade: Filtre por nível de impacto
    • Equipamento: Veja modos de falha associados a um equipamento específico
  3. Use a Pesquisa para encontrar modos de falha por palavra-chave
  4. Clique em qualquer modo de falha para ver sua descrição completa, causas prováveis e soluções recomendadas

Esta biblioteca serve como base de conhecimento para treinamento e consulta rápida durante o diagnóstico de problemas.

Melhores Práticas: Estabeleça um processo de revisão periódica dos modos de falha cadastrados, atualizando-os com base nas experiências recentes e adicionando novos conforme necessário. Uma biblioteca de modos de falha bem documentada é um ativo valioso para a gestão do conhecimento técnico da organização.

8. Categorias de Modos de Falha

O módulo de Categorias de Modos de Falha permite agrupar e classificar os diferentes modos de falha em categorias lógicas, facilitando a organização, o acesso e a análise das informações.

8.1 Categorias Pré-definidas

O sistema oferece algumas categorias padrão para classificação dos modos de falha:

Categoria Descrição Exemplos de Modos de Falha
Mecânica Falhas relacionadas a componentes mecânicos móveis Desgaste, fratura, deformação, desalinhamento
Elétrica Falhas no sistema elétrico ou eletrônico Curto-circuito, sobrecarga, mau contato
Hidráulica Falhas em sistemas que transportam fluidos Vazamento, entupimento, cavitação
Pneumática Falhas em sistemas que utilizam ar comprimido Perda de pressão, vazamento de ar, contaminação
Estrutural Falhas na estrutura ou base do equipamento Rachadura, corrosão, fadiga estrutural
Instrumentação Falhas em instrumentos de medição e controle Descalibração, leitura incorreta, falha de sensor

8.2 Criando Nova Categoria de Modo de Falha

  1. No menu lateral, acesse Configurações e depois Categorias de Modos de Falha
  2. Clique no botão Nova Categoria
  3. Preencha os campos necessários:
    • Nome da Categoria: Identificação clara e concisa
    • Descrição: Explicação detalhada do tipo de falha
    • Cor: Escolha uma cor para identificação visual da categoria
  4. Adicione Subcategorias, se necessário
  5. Clique em Salvar para criar a nova categoria

8.3 Filtrando e Analisando por Categoria

Para utilizar as categorias em análises e relatórios:

  • No módulo de Relatórios, selecione Análise de Modos de Falha
  • Utilize o filtro Categoria para restringir a análise a um tipo específico
  • Compare a frequência de ocorrência entre diferentes categorias
  • Identifique quais categorias de falha causam maior impacto na disponibilidade dos equipamentos
  • Analise tendências de aumento ou redução em categorias específicas ao longo do tempo

As categorias permitem análises mais direcionadas e a identificação de áreas críticas que requerem ações prioritárias.

8.4 Hierarquia de Categorias

O sistema permite criar uma estrutura hierárquica de categorias e subcategorias:

  1. Para adicionar uma subcategoria, acesse a página de detalhes da categoria principal
  2. Clique em Adicionar Subcategoria
  3. Preencha as informações da subcategoria
  4. Clique em Salvar

Exemplo de hierarquia:

  • Mecânica (Categoria principal)
    • Rolamentos (Subcategoria)
    • Engrenagens (Subcategoria)
    • Correias (Subcategoria)

Esta estrutura permite uma classificação mais precisa e granular dos modos de falha.

Recomendação: Desenvolva um sistema de categorias que atenda às necessidades específicas da sua organização e dos tipos de equipamentos que você gerencia. Uma classificação bem estruturada facilita a identificação de padrões e a implementação de melhorias nos processos de manutenção.

9. Causas de Falha

O módulo de Causas de Falha permite identificar e documentar as origens ou razões fundamentais que levam aos modos de falha, facilitando a implementação de ações corretivas e preventivas eficazes.

9.1 Diferenciando Modos e Causas de Falha

É importante entender a diferença entre modos de falha e causas de falha:

  • Modo de Falha: É a maneira como a falha se manifesta, ou seja, a forma como o equipamento apresenta o problema (ex: vazamento, quebra, superaquecimento)
  • Causa de Falha: É o fator que originou o modo de falha, ou seja, a razão subjacente que levou ao problema (ex: corrosão, fadiga de material, erro operacional)

Por exemplo, um modo de falha pode ser "Rompimento de uma mangueira hidráulica", enquanto a causa de falha pode ser "Desgaste por abrasão devido ao atrito com uma superfície metálica".

9.2 Cadastrando uma Nova Causa de Falha

  1. No menu lateral, acesse Configurações e depois Causas de Falha
  2. Clique no botão Nova Causa de Falha
  3. Preencha os campos do formulário:
    • Nome da Causa: Identificação concisa
    • Descrição Detalhada: Explicação completa do mecanismo de falha
    • Categoria: Selecione a categoria da causa de falha
    • Detectabilidade: Facilidade de identificação (Fácil, Média, Difícil)
    • Modos de Falha Associados: Quais modos de falha podem ser ocasionados por esta causa
  4. Adicione Ações Preventivas Recomendadas para evitar esta causa de falha
  5. Clique em Salvar para registrar a nova causa de falha

9.3 Análise de Causa Raiz

O sistema permite realizar uma análise aprofundada para identificar a causa raiz de problemas recorrentes:

  1. Na página de Manutenção Corretiva, selecione um registro e clique em Análise de Causa Raiz
  2. Utilize a ferramenta de 5 Por quês:
    • Por que ocorreu o problema? (1º nível)
    • Por que ocorreu esta causa? (2º nível)
    • Por que esta condição estava presente? (3º nível)
    • Por que não foi detectado antes? (4º nível)
    • Por que não existem controles? (5º nível)
  3. Para cada resposta, registre a causa correspondente e classifique-a
  4. Crie um Plano de Ação para eliminar a causa raiz identificada
  5. Estabeleça Indicadores de Acompanhamento para verificar a eficácia das ações

Esta metodologia estruturada permite identificação sistemática das causas fundamentais e sua eliminação permanente.

9.4 Matriz de Relação Causa-Efeito

O sistema gera automaticamente uma matriz que relaciona causas e modos de falha:

  1. Acesse Relatórios > Matriz Causa-Efeito
  2. Selecione o Período e o Equipamento a ser analisado
  3. A matriz mostrará quais causas estão mais frequentemente associadas a determinados modos de falha
  4. Identifique as causas críticas que provocam múltiplos modos de falha
  5. Priorize ações preventivas nas causas mais impactantes

Esta análise permite direcionar esforços para eliminar as causas que provocam o maior número de problemas.

Melhores Práticas: Ao identificar causas de falha, sempre busque a mais fundamental possível. Por exemplo, ao invés de registrar apenas "falha de lubrificação", investigue por que houve falha na lubrificação: foi por esquecimento, por erro no procedimento, por falta de treinamento ou por outra razão mais específica? Quanto mais profunda a investigação, mais eficaz será a ação corretiva.

10. Categorias de Causas de Falha

O módulo de Categorias de Causas de Falha permite classificar as diferentes origens de falhas em grupos lógicos, facilitando a identificação de padrões e a implementação de ações sistemáticas de melhoria.

10.1 Categorias Padrão de Causas de Falha

O sistema oferece algumas categorias pré-definidas para classificação das causas de falha:

Categoria Descrição Exemplos de Causas
Desgaste Natural Deterioração normal devido ao uso ao longo do tempo Fadiga de material, envelhecimento, fim de vida útil
Erro Operacional Falhas causadas por operação incorreta do equipamento Sobrecarga, operação fora de parâmetros, erro de manobra
Falha de Manutenção Problemas decorrentes de manutenção inadequada Lubrificação insuficiente, ajuste incorreto, reparo mal executado
Fatores Ambientais Causas relacionadas ao ambiente de operação Temperatura extrema, umidade, sujeira, vibração externa
Falha de Projeto Problemas oriundos de erro na concepção do equipamento Subdimensionamento, seleção inadequada de material, design impróprio
Qualidade de Material Falhas relacionadas à qualidade dos componentes Defeito de fabricação, material fora de especificação, contrafeit

10.2 Criando Nova Categoria de Causa de Falha

  1. No menu lateral, acesse Configurações e depois Categorias de Causas de Falha
  2. Clique no botão Nova Categoria
  3. Preencha os campos necessários:
    • Nome da Categoria: Identificação clara e concisa
    • Descrição: Explicação detalhada do tipo de causa
    • Cor: Escolha uma cor para identificação visual da categoria
    • Área Responsável: Departamento normalmente responsável por ações relacionadas a esta categoria
  4. Clique em Salvar para criar a nova categoria

10.3 Analisando Dados por Categoria de Causa

Para obter insights sobre as categorias de causas de falha mais frequentes:

  1. Acesse Relatórios > Análise de Causas
  2. Selecione o período desejado e o equipamento (ou grupo de equipamentos)
  3. Visualize o gráfico de distribuição de causas por categoria
  4. Identifique as categorias mais recorrentes
  5. Para cada categoria crítica, aprofunde a análise explorando as causas específicas
  6. Estabeleça planos de ação direcionados para as categorias mais impactantes

Esta análise permite implementar ações sistemáticas que abordam grupos inteiros de causas relacionadas.

10.4 Estratégias de Melhoria por Categoria

Cada categoria de causa de falha requer abordagens diferentes para mitigar problemas:

Categoria Estratégias de Melhoria
Desgaste Natural Planos de substituição preventiva, monitoramento de condição, especificação de materiais mais duráveis
Erro Operacional Treinamento, criação de procedimentos operacionais padrão, sistemas à prova de falhas (poka-yoke)
Falha de Manutenção Revisão de procedimentos, treinamento da equipe técnica, checklist de verificação
Fatores Ambientais Adequação do ambiente, proteções adicionais, seleção de componentes resistentes às condições locais
Falha de Projeto Reengenharia, modificações de projeto, envolvimento com fabricante

Ao categorizar adequadamente as causas, é possível implementar programas de melhoria abrangentes que tratam grupos inteiros de problemas similares.

Recomendação: Realize periodicamente uma análise de Pareto das categorias de causas de falha para identificar quais responsáveis pela maior parte dos problemas (geralmente seguindo o princípio 80/20, onde 20% das categorias de causas são responsáveis por 80% das falhas). Concentre seus esforços de melhoria nessas categorias críticas para maximizar o retorno sobre o investimento.

11. Relatórios de Manutenção

O módulo de Relatórios de Manutenção fornece ferramentas para análise de dados, monitoramento de desempenho e tomada de decisões baseadas em informações consolidadas sobre todas as atividades de manutenção.

Fig. 7: Interface do módulo de relatórios de manutenção

11.1 Relatórios Disponíveis

O sistema oferece diversos relatórios pré-configurados para análise de dados:

  • Resumo de Manutenções: Visão geral de todas as manutenções realizadas em um período
  • Desempenho por Equipamento: Análise de disponibilidade, confiabilidade e custos por equipamento
  • Indicadores de Manutenção: MTBF, MTTR, disponibilidade e outros KPIs
  • Análise de Falhas: Distribuição de falhas por modo, causa e categoria
  • Custos de Manutenção: Detalhamento de custos por tipo, equipamento e período
  • Histórico de Equipamentos: Registro completo de intervenções por equipamento
  • Programação e Cumprimento: Acompanhamento de planos de manutenção preventiva
  • Estoque de Peças: Consumo e disponibilidade de peças de reposição

11.2 Filtrando e Personalizando Relatórios

Para obter informações mais específicas:

  1. Acesse o módulo Relatórios
  2. Selecione o tipo de relatório desejado
  3. Utilize os filtros disponíveis para refinar os dados:
    • Período: Selecione o intervalo de datas a ser analisado
    • Equipamento/Grupo: Filtre por equipamento específico ou grupo
    • Departamento/Área: Restrinja a localização dos equipamentos
    • Tipo de Manutenção: Filtre por categoria (preventiva, corretiva, etc.)
    • Responsável: Filtre por técnico ou equipe
  4. Clique em Gerar Relatório para aplicar os filtros
  5. Use as opções de Personalização para ajustar a visualização:
    • Escolha o tipo de gráfico (barras, pizza, linha, etc.)
    • Selecione as colunas a serem exibidas nas tabelas
    • Defina o agrupamento dos dados (por mês, por categoria, etc.)
    • Ajuste a escala e as cores dos gráficos

Os relatórios personalizados podem ser salvos para uso futuro.

11.3 Exportando e Compartilhando Relatórios

Após gerar um relatório, você pode exportá-lo em diferentes formatos:

  • PDF: Para apresentações e documentos formais
  • Excel: Para análise adicional dos dados
  • CSV: Para importação em outros sistemas
  • Imagem (PNG/JPG): Para inclusão em apresentações

Para compartilhar relatórios com a equipe:

  1. Clique no botão Compartilhar
  2. Selecione o método de compartilhamento:
    • Email: Envie o relatório diretamente por email
    • Link: Gere um link para acesso ao relatório
    • Agendar: Configure o envio automático periódico
  3. Defina os destinatários e permissões de acesso
  4. Adicione comentários ou observações, se necessário

11.4 Configuração de KPIs e Dashboards

Crie dashboards personalizados com os indicadores mais relevantes para sua operação:

  1. Acesse Configurações > Dashboards Personalizados
  2. Clique em Novo Dashboard
  3. Selecione os KPIs que deseja monitorar, como:
    • Tempo Médio Entre Falhas (MTBF)
    • Tempo Médio de Reparo (MTTR)
    • Disponibilidade de Equipamentos
    • Taxa de Cumprimento de Preventivas
    • Custo de Manutenção por Valor de Reposição (%)
    • Backlog de Manutenção (horas)
  4. Configure Alertas para notificação quando indicadores ultrapassarem limites pré-definidos
  5. Defina a Periodicidade de atualização dos dados
  6. Organize os elementos no layout do dashboard usando a interface de arrastar e soltar
  7. Salve o dashboard e defina como página inicial, se desejado

11.5 Análise de Tendências e Previsões

Utilize as ferramentas de análise avançada para identificar tendências e fazer predições:

  • Análise de Tendências: Visualize a evolução dos indicadores ao longo do tempo
  • Correlação: Identifique relações entre diferentes variáveis (ex: temperatura ambiente vs. falhas)
  • Previsão de Falhas: Com base no histórico, estime quando ocorrerão as próximas falhas
  • Simulação: Teste diferentes cenários de manutenção e seus impactos
  • Análise de Confiabilidade: Avalie a confiabilidade dos equipamentos ao longo do tempo

Estas ferramentas ajudam a antecipar problemas e otimizar as estratégias de manutenção.

Dica: A análise regular dos relatórios de manutenção é fundamental para a melhoria contínua. Estabeleça reuniões periódicas para revisar os indicadores, identificar oportunidades de melhoria e definir ações corretivas. Lembre-se de que dados de qualidade são essenciais para análises confiáveis, portanto, incentive o registro preciso e completo de todas as atividades de manutenção.
